# 4.1 Motivated Example Figure（动机示例图）

## 论文绘图总论

在计算机科学与数据智能领域的顶会论文（如 SIGMOD, VLDB, ICML, NeurIPS 等）中，图表的质量往往直接决定了审稿人对论文的第一印象。Reviewers 在评估论文时，"Nice Presentation"（专业排版与高质量图表）是核心评价维度之一。**画一张好图可能需要 1-2 天时间，这是非常正常的投入。**

在开始讲解具体的图表类型之前，先明确几条通用的**绘图铁律**：

| 铁律 | 说明 |
|------|------|
| **必须使用矢量格式** | 导出和插入图表时，必须使用 PDF、EPS 或 SVG 格式，绝对不要使用 PNG 或 JPG 截图插入论文，放大后会模糊，这是典型的"业余"标志 |
| **字体大小要合理** | 图表中的字体（尤其是坐标轴刻度和标签）在插入到 LaTeX 双栏排版后，必须与正文字体大小相近（通常不小于 8pt）。千万不要出现图中字小到需要放大镜看的情况 |
| **画布尺寸要小** | 在 Matplotlib 中，将画布设置得小一些（例如 150x100pt），这样字体和线条在缩放后会自然变大、变粗，看起来更清晰。不要用大画布配小字体 |
| **颜色要考虑色盲友好** | 约 8% 的男性有色觉障碍。使用色盲友好的配色方案（如 ColorBrewer），并且永远不要仅依靠颜色来传递信息 |
| **图注（Caption）不可省略** | 每张图都必须有完整的图注，使图表能够独立于正文被理解（Self-contained） |

本节将重点讲解论文中最核心的三类图表。

---

## 核心目标

Motivated Example Figure 是整篇论文中**最重要的一张图**。它的使命是在读者阅读 Introduction 的前两段后，用一张图在 30 秒内直观地向读者传递两个核心信息：**"这篇论文要解决什么问题"** 以及 **"为什么现有方法不行"**。

这张图通常出现在论文的第一页右上角或第二页顶部，紧跟在 Introduction 中描述 Limitation 的段落之后。它是审稿人决定是否继续认真阅读你论文的"门面"。

## 常见的设计范式

在顶会论文中，Motivated Example Figure 主要有以下三种经典范式：

### 范式一：Running Example + Failure Case（最推荐）

这是最经典、最有说服力的范式。结合前文提到的 Running Example，给出一个真实的、具体的场景，展示现有方法在该场景下的失败。

**案例参考**：假设你研究的是 Text-to-SQL 任务。你可以在图中展示一个具体的自然语言查询（如"找出所有在 2024 年发表了至少 3 篇论文的教授及其所在院系"），然后并排展示：(a) 现有方法生成的错误 SQL（缺少了 GROUP BY 子句），以及错误的查询结果；(b) 正确的 SQL 和预期结果。用红色高亮错误部分，用绿色高亮正确部分。

### 范式二：Existing vs. Ours 对比图

将图分为左右两栏（或上下两栏）。左边展示现有方法的工作方式及其局限性，右边展示你的方法的工作方式及其优势。通过视觉上的直接对比，让读者一目了然地看到差异。

**案例参考**：AFlow（ICLR 2025）的 Figure 2 展示了 Node、Operator 和 Edge 三个核心概念的具体结构，让读者直观理解这套新的工作流表示方式与传统方法的本质区别。

![AFlow Figure 2：Existing vs. Ours 对比图示例](../../assets/images/aflow_existing_vs_ours.png)

### 范式三：性能预览图（Performance Teaser）

在 Introduction 中直接放一张精心设计的性能对比图，让读者第一眼就看到你的方法的效果。这种范式适合你的方法在性能上有显著优势的情况。

**案例参考**：Alpha-SQL（ICML 2025）的 Figure 1 就是一个典型的 Performance Teaser。它用柱状图展示了 Alpha-SQL 作为 plug-in 框架，在不同规模的开源 LLM 上均能带来显著增益，并直观传达出"小模型也能被明显增强"这一核心卖点。

![Alpha-SQL Figure 1：性能预览图示例](../../assets/images/performance_teaser_example.png)

## 设计原则

- 先在纸上或白板上画草图，与导师讨论确认后再上机绘制，这能避免大量的返工。
- 也可以多参考本教程 Vibe Research 的章节，分享了如何辅助绘图。

## 工具推荐

Motivated Example Figure 通常包含文字、图标、箭头、代码片段等混合元素，因此推荐使用 **PowerPoint (PPT)** 或 **Figma** 来绘制。PPT 的优势在于上手快、排版灵活，绘制完成后导出为 PDF 即可获得矢量图。Figma 的优势在于协作方便、组件化设计。对于包含代码片段的图，可以先用代码高亮工具生成截图，再在 PPT/Figma 中排版。
